# Task: Mock API Enhancement - Data Generators and Service Layer

## Description

Extend the existing mock API layer (`omeApi.ts` and `mockData.ts`) to generate realistic data for devices, alerts, jobs, and reports. Implement OData-style query parameter support, simulated network delays, and error scenarios. This provides the data foundation for all UI features without requiring backend integration.

## Acceptance Criteria

- [ ] `generateDevices()` creates 10-50 mock devices with realistic attributes (name, service tag, model, IP, status, type)
- [ ] `generateAlerts()` creates Critical, Warning, and Info alerts with timestamps, categories, and recommended actions
- [ ] `generateJobs()` creates Running, Completed, and Failed job states with progress percentages
- [ ] `generateReportData()` creates CSV-ready report structures for all 6 report types
- [ ] API service layer supports OData query parameters ($filter, $top, $skip, $orderby)
- [ ] Simulated network delays (100-500ms) for realistic behavior
- [ ] 5% random failure rate in dev mode for error handling testing
- [ ] Alert acknowledgment action with optimistic updates
- [ ] TypeScript interfaces for all data models (Device, Alert, Job, Report)

## Technical Details

**Implementation Approach:**
- Extend `ome-dashboard/src/services/mockData.ts` with data generators
- Update `ome-dashboard/src/services/omeApi.ts` with new endpoints
- Create `ome-dashboard/src/types/ome.ts` for TypeScript interfaces
- Use faker.js or similar library for realistic mock data generation
- Implement query parameter parsing for filtering and sorting

**Key Considerations:**
- Generate realistic Dell server model numbers (PowerEdge R640, R740, etc.)
- Use realistic IP address ranges for management networks
- Create time-based alert timestamps (last 24-48 hours)
- Simulate job progress updates for running jobs
- Ensure data consistency across related entities

**Code Locations:**
- `ome-dashboard/src/services/mockData.ts` (extend existing)
- `ome-dashboard/src/services/omeApi.ts` (extend existing)
- `ome-dashboard/src/types/ome.ts` (create new)
